# Bank REST API

Учебный REST API для управления пользователями, банковскими картами и переводами средств.  
Проект выполнен в рамках тестового задания.

---

## Технологии

- Java 17
- Spring Boot
- Spring Web
- Spring Data JPA
- Spring Security (JWT)
- PostgreSQL
- Liquibase
- Maven
- JUnit 5 / Mockito

---

## Функциональность

### Аутентификация и пользователи
- Регистрация пользователя
- Аутентификация по логину и паролю
- JWT-аутентификация
- Роли пользователей: `USER`, `ADMIN`

### Банковские карты
- Создание банковской карты
- Получение списка карт (с пагинацией)
- Блокировка и разблокировка карты
- Просмотр карт:
    - пользователь — только свои карты
    - администратор — все карты

### Переводы
- Переводы между картами

---

## Архитектура

Проект построен по классической многослойной архитектуре:

- Controller — REST API
- Service — бизнес-логика
- Repository — работа с базой данных
- DTO — обмен данными между слоями
- Entity — JPA-сущности

---

## Безопасность

- Используется JWT-токен для аутентификации
- Доступ к эндпоинтам ограничен ролями
- Пароли обрабатываются упрощённо

В рамках тестового задания криптография и практически не реализована.

---

## База данных

- PostgreSQL
- Миграции выполняются через Liquibase
- Все изменения схемы описаны в `db/migration`

---

## Тестирование

В проекте реализованы:
- Unit-тесты сервисов
- Тесты REST-контроллеров

Тесты покрывают основные сценарии работы приложения.
